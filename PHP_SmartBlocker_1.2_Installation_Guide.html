PHP SmartBlocker Installation Guide v1.2

PHP SmartBlocker is a zero-dependency PHP bot protection system featuring a drag-and-drop puzzle CAPTCHA, automatic log rotation, and zero external dependencies — all running locally on your server with no external APIs or paid services.

✓ No API calls ✓ No paid services ✓ Zero ongoing cost ✓ Cloudflare-aware PHP 7.4+ 2 files to upload Shared hosting friendly

How It Works

One line in .htaccess tells PHP to run blocks.php before every single page on your site. Every request is filtered in this order:

StepCheckAction
1Empty user agentHard 403 — no UA means definitely a bot
2Known good botsAllow silently (Googlebot, Bingbot etc.)
3Known bad bot UAsHard 403 — curl, scrapy, selenium etc.
4Old Chrome (<110)Hard 403 — bot fingerprint
5Puzzle CAPTCHADrag-and-drop challenge for all unverified visitors
✅ Once solved, a 24-hour cookie lets the visitor through freely. Bots can never solve the puzzle. Any spammer that gets through is a real human — not a bot.
ℹ Cloudflare-aware, works either way. blocks.php resolves the visitor's real IP itself — if your server's direct connection comes from a verified Cloudflare IP range, it trusts the CF-Connecting-IP header for the true visitor address; otherwise it uses REMOTE_ADDR exactly as before. You don't need to configure anything, and nothing changes if you never put the site behind Cloudflare.

Visitor Experience

1
First visit — Drag-and-drop puzzle appears. Takes about 5 seconds to solve.
2
After solving — 24-hour cookie set. Visitor browses freely with no further interruption.
3
Next day — Cookie expires. Puzzle shows once more on first visit of the day.
4
Wrong answer — puzzle resets with a new random theme and slot. They can try again immediately.

Files Required

Upload both PHP files to the same directory as your website root.

⚙️
blocks.php
Main bot blocker. Runs before every page via .htaccess auto-prepend. Handles all filtering, logging, and CAPTCHA routing.
🧩
verify_overlay.php
The drag-and-drop puzzle CAPTCHA page. 5 random landscape themes, random missing slot position. Resets to a new random puzzle on a wrong answer. Sets 24-hour cookie on success.
ℹ alist.txt is created automatically on first run. You do not need to create them.

Installation Steps

⚠ Paths — change to match your server. All paths in this guide use /home/yourusername/public_html/ as an example. Replace yourusername with your own cPanel username. Find your exact path in cPanel → File Manager → navigate to public_html → the address bar shows your full path.
1

Upload the PHP Files

Upload both files to your website root:

Upload paths
/home/yourusername/public_html/blocks.php
/home/yourusername/public_html/verify_overlay.php
⚠ Change yourusername to your own cPanel username before uploading. For example if your username is johndoe your path would be /home/johndoe/public_html/. Find your exact path in cPanel → File Manager → the address bar shows the full path when you open public_html.
ℹ Both files must be in the same folder. blocks.php uses __DIR__ to find verify_overlay.php.
2

Edit Your .htaccess File

Open .htaccess in your website root and add this line. Create the file if it does not exist.

.htaccess
php_value auto_prepend_file /home/yourusername/public_html/blocks.php
⚠ Replace yourusername with your own cPanel username to match your server path.
⚠ If you already have an auto_prepend_file line, replace it — do not have two.
3

Verify File Permissions

The website root needs write permission so PHP can auto-create the log file.

ℹ On shared hosting (cPanel) your public_html is typically already set to 755. No action needed unless you see permission errors.
4

Open your website in an incognito/private window. You should immediately see the drag-and-drop puzzle.

What's New in v1.2

  
⚠ Supersedes the v1.1 "IP source unified" change below. v1.1 made IP resolution REMOTE_ADDR-only everywhere to keep both files in sync. v1.2 replaces that with Cloudflare-aware resolution in both files, using the same shared logic so they stay in sync either way.

  

    
ChangeFileDetails

    
      
Cloudflare-aware IP resolutionblocks.phpNew getRealIp() checks whether the direct connection (REMOTE_ADDR) is a verified Cloudflare IP (full published range list included). If so, it trusts CF-Connecting-IP for the real visitor address; otherwise it falls back to REMOTE_ADDR unchanged. A random visitor can't spoof this by sending a fake header directly, since the header is only trusted when it actually arrives via Cloudflare's network.

      
verify_overlay.php IP fixverify_overlay.phpoverlay_get_ip() now calls the same getRealIp() function from blocks.php (already in scope, since this file is required from inside blocks.php) instead of reading REMOTE_ADDR on its own. Without this, the IP stored in $_SESSION['verified_ip'] after solving the CAPTCHA could mismatch what blocks.php checks on the next request once Cloudflare is added — causing the puzzle to reappear on every page load.

Troubleshooting

  

    
ProblemSolution

    
      
Puzzle never showsCheck .htaccess has the auto_prepend_file line. Verify blocks.php is uploaded to the correct directory.

      
500 error on all pagesCheck PHP error log. Usually a missing file. Verify both PHP files uploaded completely without truncation.

      
Drag does not workOpen browser console for JS errors. Ensure verify_overlay.php was not truncated during upload.

      
Solved puzzle, still redirectedCheck PHP sessions are working on your host. Check cookies are not blocked by browser settings.

      
Googlebot getting puzzleCheck the allowedBots array in blocks.php includes 'googlebot'.

      
            
AUTOPOSTER_TOKEN not workingThe token is no longer hardcoded. Add SetEnv AUTOPOSTER_TOKEN yourtoken to your .htaccess or httpd.conf, matching what your autoposter sends in X-Autoposter-Token.

      
CSRF mismatch error on CAPTCHASession likely expired mid-form. Refresh the page to get a fresh puzzle and CSRF token, then try again.

      
Puzzle locked / too many attemptsWait 60 seconds. Lockout is stored in the session — clearing cookies alone won't help. Wait for the timer then refresh.

      
AJAX requests breakingAdd your endpoint paths to the $skipCaptcha array in blocks.php.

Bonus: robots.txt (Optional, Recommended)

  
This is not part of SmartBlocker itself and requires no code changes — blocks.php and verify_overlay.php remain the two-file, one-line install described above and do all the actual enforcement regardless of what's in robots.txt. This is a free, separate, optional companion step.

  
robots.txt is a plain text file that politely asks well-behaved crawlers not to visit your site. It has no enforcement power — a bot can simply ignore it. SmartBlocker remains your real security layer either way. But several major crawlers do honor it, and adding it costs nothing.

  
⚠ Reputation for robots.txt compliance varies by bot and isn't guaranteed. Test on your own site and compare your alist.txt log before and after — don't assume results will match what's reported elsewhere on the internet.
